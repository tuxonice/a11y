# 1.3.1 Info and Relationships (Level A)


Uses a table for layout purposes, which is not semantically correct. The text "Welcome to Our Website" 
is visually styled to look like a heading, but it is not marked up as a heading in the HTML.

## Bad Implementation

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Bad Implementation</title>
</head>
<body>
    <table>
        <tr>
            <td style="font-size: 24px; font-weight: bold;">Welcome to Our Website</td>
        </tr>
        <tr>
            <td>Our mission is to provide quality services.</td>
        </tr>
    </table>
</body>
</html>

```

## Good Implementation

Uses semantic HTML elements like `<header>`, `<h1>`, and `<p>` to convey the structure and meaning of the content. 
The `<h1>` element is used for the main heading, which helps screen readers and other assistive technologies understand 
the hierarchy and relationships of the content.

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Good Implementation</title>
</head>
<body>
    <header>
        <h1>Welcome to Our Website</h1>
    </header>
    <main>
        <p>Our mission is to provide quality services.</p>
    </main>
</body>
</html>

```
